The Words and Characters of Cheers

CS-215 Final Project by Augustus LaFortune

An Introduction to Cheers

Cheers was a sitcom tv show that ran for 11 years from 1982 to 1993, had 11 seasons, and 270 episodes. The show takes place in a small bar of the same name in downtown Boston. As is typical of a sitcom, the show is often formulaic and the main cast of characters each have strong personalities that don't change a lot throughout the whole series.

Having recently finished watching all 11 seasons of the show, the characters are still strong in my mind. Are the characters' strong personalities and mannerisms represented in data from the show? To start, I needed to find the data.

Obtaining, Combining, and Cleaning the Data

I obtained data from two different sources. The first source was a TV show transcripts website called Subslikescript which gave me the transcripts of all episodes in the series. I wrote my own Java code that uses the Jsoup library to download the individual lines directly from the website. In this step I recorded the line, the season and episode number, and the name of the episode. While exploring the data, I noticed that here were some errors such as the letter "l" occasionally being in places where the letter "I" should be. I wrote my own Java code that fixed many of these errors using search and replace while being conscious of context. I also got information from IMDb using Jsoup and my own Java code. This was more difficult however because the website doesn't just use html to load the page. I was able to easily download the loaded page as html and then parse and clean the information from there. This is how I got the information on air date, rating, and number of votes for that rating. I then wrote some more java code that combined these two datasets into a single csv file (Cheers_Lines.csv) with 111,697 rows and 7 columns. At this point there is one line on each row of the file, and the columns are organized as: line text, season number, episode number, episode title, air date, rating (out of 10), and the number of people that voted for that rating.

I made the decision to use semicolons as the delimiter of the file instead of the more widely used comma. This is because commas are used extensively in the dialogue, and it would be rather disruptive to remove them from the text entirely. While semicolons did appear in the dialogue, they did so much less often. Therefore, I decided to replace each semicolon in the dialogue with a comma. This does not completely preserve the grammatical integrity of the text, but I deemed it the better option when compared to removing all commas. For consistency, all .csv files in this project use ";" as the delimiter.

Data Analysis

Part 1: How can we find characters if we don't have speaker names?

One limitation with this dataset is that I do not know which character is speaking each line. This means I can not do specific character dialogue analysis like seeing if Norm says the word "beer" more than the average character on the show. This does not mean, however, that I have no way of investigating the correlation of any given character and word. Since people often speak the name of the person that they are talking to, I am able to use that as a proxy for a character's relevance to a given conversation. Look at the following exchanges to see how this works.

Example 1-a:
# Line (S11E8)
1 Well, whatever. Congratulations.
2 Yeah, you know, when, uh, Woody Boyd talks, people listen.
3 Matter of fact, Sam, I'm gonna do you a great, big favor.
4 Mr. Peterson, I'm gonna talk and you're gonna listen.
5 It's about your beer tab.
6 Wood, stay in the shallow end.
7 Yeah, maybe you're right, sorry.

We can see that the word "beer" is spoken in line #5, though we don't know by whom. Based on line #6 we can suspect that it was spoken by Woody, but we can't be sure. We do know however, that Norm's name (Norm's full name is "Norman Peterson") is spoken in line #4, just one line before the word "beer" appears. This concretely places Norm's name close to a conversation that mentions "beer."

Example 1-b:
# Line (S11E18)
1 This brings it all back.
2 It's like I never left.
3 Uh, could I have a beer, please, Gus?
4 Sure, Norm.
5 Something's different.
6 Gus is back!

We can see that the word "beer" is spoken in line #3. We again don't know who spoke the word, but this time we could surmise that it was spoken by Norm. This time there is also a name that appears in the same sentence. At this point however, we are only interested in Norm's relationship with beer. We can see that Norm's name is spoken in line #4, just one line after the word "beer" appears. Again, this concretely places Norm's name close to a conversation that mentions "beer," regardless of speaker.

I chose to consider a character relevant to a given line if the character's name was spoken within two lines of it. This is an arbitrary range that I chose based on the nature of conversations that I observed. This means that in the first example above, Norm (mentioned in line #4) would be considered relevant for lines #2-6. Since the word "beer" is spoken in line #5, this instance of the word "beer" is considered to be related in some way to the character Norm.

I created a new file (Cheers_Name_Index.csv) that, for each of the 15 characters, and each of the 111,697 lines of dialogue, indicated if the character was mentioned near that line. When performing this step, I made sure to be aware of other names the character may go by and to count those too. (For example, the character Sam also uses the name "Malone," "Sammy," or "Mayday.") The character index data was simply stored as a boolean. Either the character was mentioned nearby (true / 1), or they were not (false / 0). Look at Example 2 to see how this works. Look at the new character columns for this same exchange we looked at in Example 1-a. (For the sake of clarity, the two lines above and below each name mention is not included in this example index. In the Cheers_Name_Index.csv file, however, the nearby lines are included.)

Example 2:

This data was saved to the Cheers_Name_Index.csv file in order to avoid needing to recheck each string for every name every time the code is run. Next, I created a similar file that contained the index of every occurrence of the word "beer." (This time not including the nearby lines, because unlike with the character names there is no uncertainty as to when beer is relevant in the conversation. It is spoken directly when it is relevant.) Given these two files, I was able to easily calculate the correlation between each character and the word "beer." But before we look at that, let's find out what other words might bring up interesting correlations.

Part 2: How can we find interesting words?

There are multiple common ways to explore word frequency in a large body of text. The first method that I decided to use is called TF-IDF. This stands for Term Frequency - Inverse Document Frequency. A high weighted TF-IDF is the result of a high term frequency in a given document and a low document frequency of the term in the whole collection of documents. The idea of this is to measure how important a given term is by weighting the document frequency by the specificity of the term. For example, in most documents the word "the" would have the highest document frequency as it is the most often word used in the English language. This also means that it has a low specificity, which lowers its importance.

Using TF-IDF on this dataset did not give very meaningful results. There were dozens of words that were tied for the same score. This is because, even with such a large dataset, there were many words that appeared the exact same number of times as each other. The following table shows the first five words in the list of all terms used. Each of these words was spoken nine times, in nine different lines throughout the whole tv show.

Term TF-IDF Value
absolute 10.22564
adults 10.22564
advertising 10.22564
affairs 10.22564
airplane 10.22564

Unsatisfied with these results, I started to look for a method that would tell me which words are used uniquely often in the tv show. This search led me to using a Log-Likelihood test against a reference corpus. This would tell me how the word frequencies in my dataset compared to the reference. I decided to use a dataset called the SUBTLEXus corpus as the reference. This dataset of word frequency is based on the subtitles of English and American movies and tv shows. This makes it a more sensible choice when compared to many other datasets, such as the Brown Corpus, which are based on written documents or are far out of date.

Let's look at the top five words with the highest LL score. (The five most overrepresented words in the tv show.)

Term LL Score Count
sam 27773.17 4217

Even just looking briefly we can see that most of the top words are the names of the main characters. This makes sense because a character name is something that is relatively unique. The character Sam Malone from Cheers is far from the only Sam that appears in all the movies and tv shows included in the SUBTLEXus corpus, but the name Sam is far from being as prolific as most English words. We finally have a list of "Cheers keywords." These words appear uniquely often in the show. I saved this list to another .csv file (Cheers_Clean_Keywords.csv). I find this list interesting because while watching the show, I did not think about the frequency of these words. Now that I know that the word "well" is used a lot, it is easy to find an example of a conversation that demonstrates the frequent word use. The following is just one example I found.

# Line (S5E7)
1 Well, this happens to be Sam Mayday Malone, formerly of the Red Sox.
2 Well, yeah, that'll be fine.
3 8:00? Well, let's say 8:00.

While this list is more meaningful, it still does not give us much insight into what defines each individual character. What if, for each of the top keywords, we find its correlation with each of the top characters, and then rank the words by how strong their correlation for any single character is? These values are stored in a new file (Cheers_Character-Keyword_Correlation.csv). Let's look at the top of that list.

Word Character Correlation P_value
miss Rebecca 0.124324 0.000000e+00

afternoon Norm 0.058100 4.011009e-84
beer Norm 0.056659 4.342002e-80

Wow! Each of these word-character pairs can be explained by memorable character interactions in the show. While the correlations are weak, the p-values are extremely low. This means that the correlation almost certainly indicates a real correlation in the data.

Lots of Plots!

To make it easier to see every character's relation to a given word, lets create a bar chart. On this plot the vertical axis is correlation and the different characters are spread along the horizontal axis. If the p-value of the correlation between a given character and word is not significant, that is to say, if it is greater than 0.05, the bar will appear greyed out. You can select any word from the dropdown menu (this includes the top 400 or so keywords for the show) to explore different plots. You can also hover your cursor over any of the bars to see exact numbers for correlation and p-value.
