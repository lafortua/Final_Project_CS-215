The Words and Characters of Cheers

CS-215 Final Project by Augustus LaFortune

An Introduction to Cheers

Cheers was a sitcom tv show that ran for 11 years from 1982 to 1993, had 11 seasons, and 270 episodes. The show takes place in a small bar of the same name in downtown Boston. As is typical of a sitcom, the show is often formulaic and the main cast of characters each have strong personalities that don't change a lot throughout the whole series.

Having recently finished watching all 11 seasons of the show, I wondered if the strong personalities can actually be captured by concrete data analysis. To start, I needed to find the data.

Obtaining, Combining, and Cleaning the Data

I obtained data from two different sources. The first source was a TV show transcripts website which gave me the transcripts of all episodes in the series. I wrote my own Java code that uses the Jsoup library to download the individual lines directly from the website. In this step I recorded the line, the season and episode number, and the name of the episode. While exploring the data, I noticed that here were some errors such as the letter "l" occasionally being in places where the letter "I" should be. I wrote my own Java code that fixed many of these errors using search and replace while being conscious of context. I also got information from IMDb using Jsoup and my own Java code. This was more difficult however because the website doesn't just use html to load the page. I was able to easily download the loaded page as html and then parse and clean the information from there. This is how I got the information on air date, rating, and number of votes for that rating. I then wrote some more java code that combined these two datasets into a single csv file with 111,697 rows and 7 columns. At this point there is one line on each row of the file, and the columns are organized as: line text, season number, episode number, episode title, air date, rating (out of 10), and the number of people that voted for that rating.

I made the decision to use semicolons as the delimiter of the file instead of the more widely used comma. This is because commas are used extensively in the dialogue, and it would be rather disruptive to remove them from the text entirely. While semicolons did appear in the dialogue, they did so much less often. Therefore, I decided to replace each semicolon in the dialogue with a comma. This does not completely preserve the grammatical integrity of the text, but I deemed it the better option when compared to removing all commas.

Data Analysis

Part 1: How can we find characters if we don't have speaker names?

One limitation with this dataset is that I do not know which character is speaking each line. This means I can not do specific character dialogue analysis like seeing if Norm says the word "beer" more than the average character on the show. This does not mean, however, that I have no way of investigating the correlation of any given character and word. Since people often speak the name of the person that they are talking to, I am able to use that as a proxy for a character's relevance to a given conversation. Look at the following exchanges to see how this works.

Example 1:
# Line
1 Well, whatever. Congratulations.
2 Yeah, you know, when, uh, Woody Boyd talks, people listen.
3 Matter of fact, Sam, I'm gonna do you a great, big favor.
4 Mr. Peterson, I'm gonna talk and you're gonna listen.
5 It's about your beer tab.
6 Wood, stay in the shallow end.
7 Yeah, maybe you're right, sorry.

We can see that the word "beer" is spoken in line #5, though we don't know by whom. Based on line #6 we can suspect that it was spoken by Woody, but we can't be sure. We do know however, that Norm's name (Norm's full name is "Norman Peterson") is spoken in line #4, just one line before the word "beer" appears. This concretely places Norm's name close to a conversation that mentions "beer."

Example 2:
# Line
1 This brings it all back.
2 It's like I never left.
3 Uh, could I have a beer, please, Gus?
4 Sure, Norm.
5 Something's different.
6 Gus is back!

We can see that the word "beer" is spoken in line #3. We again don't know who spoke the word, but this time we could surmise that it was spoken by Norm. This time there is also a name that appears in the same sentence. At this point however, we are only interested in Norm's relationship with beer. We can see that Norm's name is spoken in line #4, just one line after the word "beer" appears. Again, this concretely places Norm's name close to a conversation that mentions "beer," regardless of speaker.

I chose to consider a character relevant to a given line if the character's name was spoken within two lines of it. This is an arbitrary range that I chose based on the nature of conversations. This means that in the first example above, Norm, mentioned in line #4, would be considered relevant for lines #2-6. Since the word "beer" is spoken in line #5, this instance of the word "beer" is considered to be related in some way to the character Norm.

I created a new file (Cheers_Name_Index.csv) that, for each of the 15 characters, and each of the 111,697 lines of dialogue, indicated if the character was mentioned near that line. When performing this step, I made sure to be aware of other names the character may go by and to count those too. (For example, the character Sam would also answer to "Malone," "Sammy," or "Mayday.") The data was simply stored as a boolean. Either the character was mentioned nearby, or they were not. Next, I created another file that contained the index of every occurrence of the word "beer." Given these two files, I was able to easily calculate the correlation between each character and the word "beer." But before we look at that, let's find out what other words might bring up interesting correlations.

Part 2: How can we find interesting words?

There are multiple common ways to explore word frequency in a large body of text. The first method that I decided to use is called TF-IDF. This stands for Term Frequency - Inverse Document Frequency. A high weighted TF-IDF is the result of a high term frequency in a given document and a low document frequency of the term in the whole collection of documents. The idea of this is to measure how important a given term is by weighting the document frequency by the specificity of the term. For example, in most documents the word "the" would have the highest document frequency as it is used very often. This also means that it has a low specificity, which lowers its importance. Using TF-IDF did not give very meaningful results. There were dozens of words that were tied for the same score. This is because, even with such a large dataset, there were many words that appeared the same number of times as each other.

The following table shows the first five words in the list of all terms used. Each of these words was spoken nine times, in nine different lines throughout the whole tv show's existence.

Unsatisfied with these results, I started to look for a method that would tell me which words are used uniquely often in the tv show. This search led me to using a Log-Likelihood test against a reference corpus. This would tell me how the word frequencies in my dataset compared to the reference. I decided to use a dataset called the SUBTLEXus corpus. This dataset of word frequency is based on the subtitles of English and American movies and tv shows. This makes it a more sensible choice when compared to many other datasets, such as the Brown Corpus, which are based on written documents.

Let's look at the top five words with the highest LL score. (The five most overrepresented words in the tv show.)

Even just looking briefly we can see that most of the top words are the names of the main characters. This makes sense because a character name is something that is relatively unique. The character Sam Malone from Cheers is far from the only Sam that appears in all the movies and tv shows included in the SUBTLEXus corpus, but the name Sam is far from being as prolific as the word "the."
